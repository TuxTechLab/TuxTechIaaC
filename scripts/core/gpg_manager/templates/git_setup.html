4. Test GPG-Signed Commit

Follow these steps to test your GPG key with a real commit:

Step 1: Fork the Test Repository
  1. Go to TuxTechLab/test-gpg-sign
  2. Click the "Fork" button in the top-right corner
  3. Select your personal account as the destination
Step 2: Clone and Configure
# Replace YOUR-USERNAME with your GitHub username
git clone git@github.com:YOUR-USERNAME/test-gpg-sign.git
cd test-gpg-sign

# Verify the remote is set to your fork
git remote -v  # Should show your fork as 'origin'

# Add the original repository as 'upstream'
git remote add upstream 'https://github.com:YOUR-USERNAME/test-gpg-sign.git'

# Verify both remotes are set correctly
git remote -v
Step 3: Make a Test Commit
# Create a test file
echo "# GPG Test Commit by $(git config user.name)" > gpg-test-$(date +%s).txt

# Add and commit with GPG signing
git add .
git commit -S -m "Test GPG signed commit"

# Push to your fork
git push origin main
Step 4: Create Pull Request

Follow these steps to create your pull request:

  1. Go to your forked repository on GitHub
  2. Click on "Pull requests" tab, then "New pull request"
  3. Set base repository to TuxTechLab/test-gpg-sign and base: main
  4. Set head repository to your-username/test-gpg-sign and compare: main
  5. Click "Create pull request"
  6. The PR template will open automatically. Please ensure you:
    • Use the title: [GPG - SUCCESS] GPG Key Setup Verification
    • Fill in your GPG key details (last 8 characters of Key ID and fingerprint)
    • Attach screenshots showing:
      • Verified commit status from GitHub
      • Your GPG key configuration
      • Local Git GPG configuration
  7. Click "Create pull request" to submit

Troubleshooting Tips

  • If you encounter branch protection rules, push your changes to a new branch first
  • If your commit doesn't show as 'Verified', ensure you've configured Git to use your GPG key
  • Verify your GPG key is added to your GitHub account in Settings > SSH and GPG keys

Alternative: You can also create an issue using the "GPG Verification" template in the Issues tab

Note: After submission, a maintainer will review your PR. This is a required step to verify your GPG setup is properly configured.

Note: You can verify your commit was signed with: git log --show-signature -1 Look for Good signature in the output.
